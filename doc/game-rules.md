# 游戏规则与数值设计（当前版，至 v3.7）

## 1. 设计目标

- 首版数值保持简单、可验证、易调整。
- 所有静态数值优先放入配置文件。
- 战斗结算必须可复现，便于排查玩家争议。

## 2. 基础属性

角色属性：

| 属性 | 说明 |
|---|---|
| hp | 生命值 |
| attack | 攻击 |
| defense | 防御 |
| attackSpeed | 攻击速度，影响在线战斗出手间隔 |
| power | 战力 |

v0.8 起职业配置扩展为三类。历史旧角色默认按 `warrior` 兼容：

- 战士 `warrior`：生命、防御和近战爆发稳定，兼容旧角色默认成长。
- 法师 `mage`：攻击成长高，防御较低，依靠术法爆发。
- 道士 `taoist`：属性均衡，偏持续伤害、治疗和后续召唤预留。

初始属性：

```text
hp = 120
attack = 12
defense = 4
attackSpeed = 100
gold = 100
```

升级成长：

```text
每级 hp + 20
每级 attack + 3
每级 defense + 1
每级 attackSpeed + 1
```

以上初始属性和升级成长为战士兼容基线；法师、道士以 `classes.json` 中的职业成长配置为准。

## 3. 经验曲线

升级所需经验：

```text
level 1 -> 2: 100
level 2 -> 3: 180
level 3 -> 4: 300
level 4 -> 5: 480
```

通用公式：

```text
requiredExp = floor(100 * level ^ 1.35)
```

首版可通过 `levels.json` 配置覆盖公式。

## 4. 战斗公式

v0.1/v0.2 采用自动回合制并一次性结算整场。v0.3 开始，在线战斗升级为会话式回合战斗，每次请求只推进一次行动。

玩家伤害：

```text
playerDamage = max(1, playerAttack - monsterDefense)
```

怪物伤害：

```text
monsterDamage = max(1, monsterAttack - playerDefense)
```

攻击速度：

```text
actionIntervalMs = max(700, 2200 - attackSpeed * 12)
```

在线回合流程：

```text
创建战斗会话
根据双方 attackSpeed 决定首个行动方和建议等待时间
前端按 suggestedDelayMs 请求下一次行动
后端推进一次玩家或怪物行动
如果一方死亡，结束战斗并结算奖励
否则记录 action，等待下一次 next 请求
```

最大回合数：

```text
maxRounds = 50
```

超过最大回合数时，按失败处理，避免异常配置导致死循环。

离线挂机不模拟每个回合，按挂机时长、地图、怪物和角色战力计算收益，最多累计 8 小时。

## 4.1 敌方阵列与技能目标

v2.5 起在线战斗以敌方阵列表达遭遇目标：

- 普通单怪战斗也会生成 1 个敌方单位。
- 自动遭遇会按数量生成多个敌方单位，敌人拥有独立生命、攻击、防御、攻速、前排/后排和存活状态。
- 玩家可手动选中一个存活敌人，普通攻击和单体技能优先命中选中目标。
- 未选择目标、目标不存在或目标已死亡时，普通攻击默认攻击第一个存活敌人。
- 敌方回合由每个存活敌人依次攻击玩家。
- 行动日志记录 `targetType` 和 `targets`，前端按实际命中目标展示，而不是只展示聚合的 `怪物 x 数量`。

当前技能目标类型：

| targetType | 说明 |
|---|---|
| single | 单体，命中当前目标 |
| front_row | 前排，命中所有存活前排目标 |
| back_row | 后排，命中所有存活后排目标；如果后排已清空，则回落到存活目标 |
| random3 | 随机多体，最多命中 3 个存活目标 |
| all | 群体，命中全部存活目标 |

## 5. 战斗奖励

胜利后结算：

- 经验
- 金币
- 掉落

失败后结算：

- 不扣除装备
- 不扣除金币
- 可记录失败日志

## 6. 掉落规则

掉落配置由 `drops.json` 管理。

单个怪物可配置多个掉落项：

```json
{
  "itemId": "item_wood_sword",
  "rate": 0.05,
  "minQuantity": 1,
  "maxQuantity": 1
}
```

首版采用独立概率判定：

```text
每个掉落项独立 roll 一次
random < rate 则掉落
```

战斗掉落遇到背包空间不足时，战斗仍正常结算；无法入包的掉落通过邮件补发，避免吞掉玩家奖励。

## 7. 装备品质

品质倍率：

| 品质 | key | 属性倍率 |
|---|---|---|
| 普通 | common | 1.00 |
| 精良 | fine | 1.20 |
| 稀有 | rare | 1.50 |
| 史诗 | epic | 2.00 |
| 传说 | legendary | 3.00 |

装备最终属性：

```text
finalStat = floor(baseStat * qualityMultiplier)
```

## 7.1 随机词条与重铸

- 非堆叠装备生成时会按品质写入随机词条，来源包括战斗掉落、任务奖励、地图奖励、邮件附件和 GM 发放。
- 当前词条可影响生命、攻击、防御、攻速或技能触发加成。
- 生命、攻击、防御和攻速词条参与角色属性与战力计算；技能触发词条参与战斗中主动技能触发概率。
- 重铸装备词条会消耗对应品质装备精华，重铸后替换原有随机词条。
- 已穿戴装备重铸后立即重算角色属性；未穿戴装备只更新该装备的展示属性。

## 8. 战力公式

首版战力：

```text
power = hp + attack * 8 + defense * 6 + attackSpeed * 2
```

穿戴装备、强化装备、激活套装奖励、装备随机词条变更后重新计算。

## 8.1 套装规则

- 装备可通过 `setId` 和 `setName` 归属到同一个套装。
- 套装奖励配置在装备的 `setBonuses` 中，按 `pieces` 设置激活件数门槛。
- 已穿戴同套装装备达到门槛后，对应生命、攻击、防御和攻速加成直接参与角色属性与战力计算。
- 当前赤月套装包含赤月刃、赤月战甲、深渊战盔、魂纹手镯、恶魔戒指。
- 赤月套装 2 件激活：攻击 +8、生命 +40。
- 赤月套装 4 件激活：攻击 +16、防御 +8、攻速 +6、生命 +100。

## 8.2 背包规则

当前背包为格子式资产管理：

- 默认容量 40 格。
- 装备每件占 1 格。
- 同类材料自动堆叠并占 1 格，第一版不限制堆叠上限。
- 已穿戴装备不显示在背包中，只显示在人物装备栏。
- 背包容量、已用格和剩余格只按未穿戴物品计算。
- 卸下装备前检查背包剩余空间，背包已满时不能卸下。
- 材料和未穿戴装备可以丢弃；已穿戴装备不能丢弃。
- 材料可以批量出售，装备批量出售后置，避免误卖。
- 未穿戴装备可以分解，已穿戴装备不能分解。
- 分解普通、精良、稀有、史诗/传说装备分别获得粗糙、精良、稀有、史诗装备精华。
- 史诗和传说装备基础返还 2 个精华；其他品质基础返还 1 个精华。
- 强化等级额外增加精华返还数量，例如 +3 普通装备返还 4 个粗糙装备精华。

满包处理规则：

- 战斗掉落：通过邮件补发无法入包的掉落。
- 任务奖励：空间不足时禁止领取，奖励不提前结算。
- 邮件附件：空间不足时禁止领取，邮件保持未领取。
- 挂机收益：空间不足时禁止领取，经验和金币不提前结算。
- 地图奖励事件：空间不足时禁止触发奖励，避免事件完成但物品未到账。
- GM 直接发物品：空间不足时阻止发放；GM 邮件发放不受影响，玩家领取时再检查空间。

## 9. 金币规则

金币来源：

- 战斗掉落
- 任务奖励
- 出售装备
- 出售材料
- GM 发放
- 邮件附件

金币消耗：

- 装备强化
- 后续背包扩容
- 后续药水购买
- 后续技能学习或升级

当前已实现装备强化金币消耗和材料消耗。v1.1 起强化规则读取 `enhancement-rules.json`：

- +1：`mat_beast_bone x1`
- +2：`mat_iron_ore x2`
- +3：`mat_iron_ore x3`、`mat_bone_shard x1`
- +4：需要装备等级段满足规则，消耗更高阶材料
- +5：当前最高强化等级，消耗赤月类材料

后续可继续按装备品质、装备等级段或活动周期增加不同强化规则。

## 9.1 世界 BOSS 规则

- 世界 BOSS 由 `world-bosses.json` 配置，包含名称、地图、怪物、刷新时间、奖励倍率、参与等级和战力门槛。
- 第一版采用多玩家异步累计伤害：玩家各自进入会话式战斗，战斗结束后把玩家攻击造成的伤害累计到当前轮次排行。
- 世界 BOSS 不直接发放普通战斗经验、金币或掉落；击败后按伤害排行通过邮件发放金币奖励。
- 世界 BOSS 被击败后进入冷却，到刷新时间后重置血量并清空上一轮伤害排行。

## 9.2 技能规则

- 技能由 `skills.json` 配置。
- 技能分主动和被动。
- 主动技能在玩家行动时可手动释放；未手动时继续按触发率和冷却自动判定。
- 主动技能可放入 1-4 号技能栏，0 表示不上栏。
- 主动技能支持伤害系数、固定伤害、破防、持续伤害和自我治疗。
- 主动技能支持 `targetType`，用于区分普通单体、前排、后排、多体随机和全体群攻。
- 被动技能永久提升角色属性，并参与战力计算。
- 技能学习和升级消耗金币和配置材料，等级、金币或材料不足时不能操作。
- 当前战斗日志会记录技能释放、持续伤害和治疗效果。

## 9.3 天赋规则

- 天赋由 `talents.json` 配置。
- 角色从 2 级开始，每提升 1 级获得 1 点天赋点。
- 天赋支持最大等级、角色等级门槛和前置天赋门槛。
- 当前天赋分为生存、输出、机动三条分支。
- 属性天赋参与角色属性和战力计算。
- 战斗天赋可提升主动技能触发率或战斗金币收益。
- 当前可消耗 100 金币重置全部天赋。

## 9.4 消耗品与战斗准备

- 消耗品属于可堆叠物品，占用规则与材料一致。
- 使用消耗品后不会直接修改角色永久属性，而是写入下一场战斗准备。
- 下一场战斗开始时，准备效果一次性加到本场战斗属性上，并在战斗开始日志中展示。
- 多瓶消耗品可以叠加；生效后清空准备状态。
- 当前药水方向包括生命、攻击、防御和攻速。

## 9.5 任务与引导规则

- 任务状态仍以进行中、可领取、已领取和锁定为主，不改变奖励领取与进度推进规则。
- 锁定任务由前置任务控制；前置完成后自动进入可推进状态。
- 任务剧情 `story` 和行动提示 `guide` 只作为展示与引导信息，不参与结算。
- 任务目标定位字段用于任务页“前往目标”、地图页目标高亮和跨地图提示；没有地图目标的等级类任务可以为空。
- 任务页推荐下一步优先展示可领取、未锁定和目标明确的任务，并可结合技能列表提示学习、装配或升级技能。
- 剧情回看和任务链总览只展示玩家已解锁或已完成的任务内容，避免提前暴露锁定剧情。

## 9.6 运营活动效果规则

- 活动由 `activities.json` 配置，只有 `status=active` 的活动效果参与结算。
- 当前效果类型包括 `battle_exp`、`battle_gold`、`drop_rate`、`idle_exp`、`idle_gold` 和 `world_boss_gold`。
- 多个 active 活动配置同类效果时，百分比按加法累加。
- 战斗经验/金币加成以基础收益为基数计算，BOSS 和自动遭遇的奖励倍率仍单独保留。
- 掉落概率加成会放大怪物掉落配置的 `rate`，最高不超过 100%。
- 挂机状态预估和挂机领取会同时返回活动加成后的总收益，以及来自活动的额外经验/金币。
- 世界 BOSS 活动金币加成只影响击败后排行奖励邮件中的金币，不改变伤害排行和 BOSS 血量。
- 榜单阶段奖励由活动 `rankingRewards` 配置，当前支持等级榜、战力榜和财富榜。
- 角色当前名次按榜单字段倒序、角色 ID 正序实时计算；达到 `maxRank` 门槛后，该活动才可领取对应榜单奖励。
- 榜单阶段奖励领取仍写入 `activity_claims`，同一角色同一活动只能领取一次。

## 9.7 公会规则

- 一个角色同一时间只能加入一个公会。
- 创建公会的角色自动成为会长。
- 普通成员可直接退出公会。
- 会长在公会成员数大于 1 时必须先转让会长，不能直接退出。
- 会长是最后一名成员时，退出会解散公会。
- 会长可踢出成员、转让会长；不能踢出自己，也不能把会长转让给非本公会成员。
- 公会捐献档位由 `guilds.json` 配置；捐献会扣除金币，增加个人贡献和公会总贡献。
- 同一捐献档位按配置限制每日次数，次数以角色和捐献档位维度统计。
- 公会商店商品由 `guilds.json` 配置；购买会校验是否加入公会、个人贡献、金币、每日限购和背包容量。
- 公会商店可同时消耗贡献和金币，成功后发放道具并写入对应流水。
- 公会活动目标由 `guilds.json` 配置，首版按公会总贡献达标；同一角色在同一公会内同一活动只能领取一次。
- 公会排行首版按公会总贡献倒序、成员数倒序和公会 ID 正序展示。

## 10. 事务规则

以下操作必须在数据库事务中完成：

- 战斗胜利结算
- 获得装备
- 金币增加
- 经验增加与升级
- 穿戴装备
- 卸下装备
- 出售装备
- 丢弃物品
- 材料批量出售
- 任务奖励领取
- 邮件附件领取
- 挂机收益领取
- 地图奖励事件触发
- 技能学习和升级
- 天赋加点和重置
- 公会创建、加入、退出、踢人、转让和解散
- 公会捐献、公会商店购买和公会活动奖励领取

原则：

```text
要么全部成功，要么全部失败。
```
