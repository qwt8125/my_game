# 文字传奇页游 PRD（当前版，至 v3.7）

本文只维护当前产品范围、已完成事实和后续方向；详细历史方案已经归档到 `doc/archive/20260520-doc-slimming/`，下一阶段执行计划以 `doc/roadmap.md` 为准。

## 1. 项目概述

本项目是一款网页文字版类传奇页游。玩家通过浏览器进行角色成长、打怪刷装、地图探索、NPC 交互、装备强化、任务推进和排行榜竞争。第一阶段目标是实现轻量可部署、核心循环完整、数据安全可备份的 MVP 版本。

## 2. 产品目标

- 提供低成本 Linux 部署能力，适合个人服务器或小型云主机运行。
- 首版优先使用 SQLite 持久化核心数据，后期可平滑接入 Redis。
- 以文字交互为主，降低美术资源依赖，突出数值成长、装备掉落和挂机收益。
- 支持后续扩展地图、怪物、装备、技能、任务、活动、公会和运营后台。

## 3. 目标用户

- 喜欢传奇、放置、文字 MUD、刷装备玩法的玩家。
- 偏好轻量网页游戏，不想下载安装客户端的用户。
- 适合碎片化时间游玩，可在线操作，也可挂机获得收益。

## 4. 核心玩法循环

```text
创建角色
  -> 进入地图
  -> 选择怪物或挂机区域
  -> 战斗获得经验、金币、材料、装备
  -> 升级、穿戴装备、提升战力
  -> 解锁更高级地图和怪物
  -> 参与排行榜、BOSS、活动等竞争内容
```

## 5. 首版功能范围

### 5.1 账号与角色

- 用户注册、登录、退出。
- 一个账号首版默认创建一个角色。
- 角色基础属性：
  - 昵称
  - 等级
  - 经验
  - 金币
  - 职业
  - 生命
  - 攻击
  - 防御
  - 攻击速度
  - 战力
  - 当前地图

当前已完成职业配置化，创建角色时可选择战士、法师、道士三类职业。战士沿用旧版基础成长以兼容已有角色，法师和道士使用独立生命、攻击、防御、攻速成长曲线；角色信息会展示职业名称、职业描述和天赋点。

当前已完成技能与天赋基础闭环：

- 技能配置化维护，按职业区分可学习技能。
- 已实现主动技能、被动技能、触发概率、冷却回合、持续伤害、破防、自我治疗和技能升级消耗。
- 技能学习/升级支持金币和材料消耗；主动技能可放入 1-4 号技能栏。
- 在线战斗在玩家回合支持手动释放技能；不手动时仍按自动触发逻辑出招。
- 在线会话式战斗中会自动判定已学习主动技能触发，并把技能冷却、持续伤害等临时状态保存在战斗会话中。
- 被动技能会直接参与角色生命、攻击、防御、攻速等属性重算。
- 天赋点按角色等级派生，当前规则为 `等级 - 1`。
- 天赋支持前置等级、最大等级、属性加成、技能触发加成和金币收益加成；当前已扩展为生存、输出、机动三条分支。
- 天赋可消耗金币重置，重置后重新分配点数。

当前已补充自动化质量保障：

- 接口 smoke 回归可覆盖注册、登录、建角、地图场景、任务剧情/行动提示/目标定位/前置链元数据、技能、天赋、排行榜、战斗、GM 配置重载、GM 发装备和装备从背包移除。
- 数值体检可检查职业对怪物战斗节奏、掉落引用、强化材料、天赋前置、地图事件引用、任务引用、装备档位、消耗品和 1-10 级等级空档。
- 打包前可使用 jar 锁诊断脚本确认本地 Spring Boot 或 IDE 进程是否占用目标 jar。

当前 1-10 级内容已补齐：

- 9 级新增赤月深渊，包含赤月守卫、深渊法师、深渊前线怪区、深渊余烬每日采集点。
- 10 级新增恶魔祭坛，包含祭坛骑士、恶魔祭司、祭坛核心怪区、恶魔核心每日采集点。
- 新增高阶装备、材料和消耗品：赤月战甲、深渊战盔、魂纹手镯、恶魔戒指、深渊余烬、恶魔核心、英雄秘药。

当前已补充在线战斗自动遭遇体验：

- 玩家在地图怪区可启动 Auto hunt。
- 每场战斗开始时从点位怪物范围内随机选择怪物。
- 遭遇数量在范围内随机，当前以一个战斗单位聚合呈现，例如 `怪物 x3`。
- 遭遇可按概率变为精英，精英会提高属性和奖励。
- 一场在线战斗结束后，前端显示休整倒计时，到点自动开始下一场；玩家可随时停止。
- 遭遇数量范围、精英概率和休整间隔已配置化到地图事件，便于不同地图形成不同战斗节奏。
- 遭遇怪物来源已支持 `encounterMonsters` 权重表，同一点位可配置不同怪物出现概率；配置加载和数值体检会校验权重、怪物存在性和地图归属。
- 在线战斗面板已从聚合怪物升级为敌方阵列：自动遭遇会展示多个敌方单位，普通攻击按单体目标推进，群攻技能按 `targetType` 展示实际命中目标。
- 当前已支持手动选中敌方单位，普通攻击和单体技能会优先命中选中目标。

### 5.3 怪物与战斗

- 战斗首版采用自动回合制，v0.3 升级为在线会话式回合战斗。
- 玩家选择地图后，可以点击普通打怪或开启挂机。
- 每次战斗结算：
  - 经验
  - 金币
  - 掉落装备
  - 掉落材料

首版战斗公式保持简单：

```text
玩家伤害 = max(1, 玩家攻击 - 怪物防御)
怪物伤害 = max(1, 怪物攻击 - 玩家防御)
胜负由双方生命值和伤害回合数决定
```

v0.1/v0.2 战斗一次性结算整场并返回完整 actions。v0.3 开始，在线战斗需要拆成战斗会话：

- `start` 创建战斗，返回 battleId 和初始状态。
- `next` 每次只推进一次行动，按攻击速度决定出手节奏。
- `status` 支持刷新页面后恢复当前战斗。
- `stream` 支持 SSE 行动推送，前端优先使用推送接收战斗行动，异常时回退 `next` 轮询。
- 战斗结束时再统一结算经验、金币、掉落、任务进度和日志。
- v2.5 起战斗响应返回 `enemies` 敌方阵列和行动 `targets`，用于区分单体攻击、前排、多体随机和群体技能。
- 离线挂机不逐回合模拟，继续按后台时间差结算收益。

战斗需要记录过程日志：

- 每次玩家攻击、怪物攻击都生成一条 action。
- 旧战斗接口返回完整 actions，供前端展示文字战斗过程；新会话接口逐条返回 action。
- 后端将整场战斗 actions 以 JSON 形式保存到 `battle_logs.actions_json`。
- 暂不拆独立 `battle_actions` 明细表，后期需要回放统计时再扩展。

装备属性：

### 5.5 背包系统

- 玩家获得的装备和材料进入背包。
- 背包支持：
  - 查看物品
  - 穿戴装备
  - 卸下装备
  - 出售装备
  - 分解未穿戴装备
  - 丢弃物品

当前已完成格子式背包视觉、道具图标、40 格容量展示、材料堆叠、丢弃、材料批量出售、强化材料消耗、装备分解、词条重铸和基础满包保护。未穿戴装备可分解为对应品质装备精华，强化等级会提高返还数量；背包与装备页会展示随机词条，并提供消耗对应品质精华的重铸入口。后续可继续支持扩展格子、仓库或更复杂的整理规则。

当前背包已支持消耗品堆叠和使用。药水类消耗品使用后会写入下一场战斗准备，战斗开始时一次性增加生命、攻击、防御或攻速，并在战斗日志中展示生效内容。

首版实现主线任务：

- 击杀指定怪物
- 达到指定等级
- 穿戴指定品质装备
- 通关指定地图
- v0.5 起建议扩展 NPC 对话、地图探索、任务前置链和地图事件任务。

任务奖励：

- 经验
- 金币
- 装备
- 材料

当前已完成 P1 任务体验增强：任务页会基于任务状态、目标位置和当前进度推荐下一步行动；可查看任务链总览并按节点切换剧情回看；任务列表提供“前往目标”和“回看剧情”入口。任务页还会读取技能列表，提示可学习、可装配或可升级的技能，帮助新手把任务推进、技能成长和战斗入口连起来。天赋重置已具备确认弹窗，避免误操作。

### 5.7 排行榜

首版排行榜可直接从 SQLite 查询：

- 等级榜
- 战力榜
- 财富榜

v0.3 已接入 Redis 短 TTL 缓存，等级榜、战力榜、财富榜优先读缓存，缓存未命中或 Redis 异常时回落 SQLite。v2.9 已补充排行榜快照接口，前端展示数据来源、生成时间、刷新间隔和下次刷新时间；旧榜单数组接口继续保留。后续玩家规模继续增长时，可再升级为 Redis Sorted Set 或定时榜单快照落库。

### 5.8 运营活动

v3.0 已补充运营活动配置入口：活动通过 `activities.json` 维护，前端活动页展示活动状态、类型、时间、说明、奖励预览和目标页面入口。v3.1 已补充活动奖励领取闭环：玩家可领取 active 活动奖励，系统写入 `activity_claims` 防重复领取，同时写入金币流水和掉落日志，并进行背包容量检查。v3.2 已补充活动效果系统，active 活动可配置战斗经验、战斗金币、掉落概率、挂机经验、挂机金币和世界 BOSS 邮件金币加成；前端活动页、战斗结算、挂机预估/领取和世界 BOSS 列表会展示对应加成。v3.3 已补充榜单阶段奖励，活动可配置排行榜类型、名次门槛和阶段奖励，活动页展示当前名次、达成状态和奖励内容，领取时复用 `activity_claims` 防重复。v3.4 已补充 GM 活动编辑后台，管理员可查看活动配置、编辑状态/时间/文案/奖励/加成/榜单奖励，保存前复用完整配置校验，保存成功后写回 `activities.json` 并立即切换运行内存快照。

### 5.9 公会系统

v3.5 已补充公会基础闭环：玩家可创建公会，创建者自动成为会长；未加入公会的玩家可查看公会列表并加入；公会页展示公会公告、会长、成员数和成员列表。会长可踢出成员、转让会长；普通成员可退出；当公会只剩会长一人时，会长退出会解散公会。公会基础数据落入 SQLite 的 `guilds`、`guild_members` 和 `guild_logs`。

v3.6 已补充公会贡献与商店闭环：公会配置通过 `guilds.json` 维护每日捐献档位和公会商店商品；玩家可每日捐献金币获得个人贡献，同时累加公会总贡献；公会商店按个人贡献、金币、每日限购和背包容量校验后发放道具。相关数据写入 `guild_donations`、`guild_shop_purchases`、`currency_logs`、`drop_logs` 和 `guild_logs`，`guilds.total_contribution` 记录公会总贡献。

v3.7 已补充公会活动与公会排行：基于公会总贡献展示公会排行，并通过 `guilds.json` 配置公会活动目标，达标后成员可领取金币和道具奖励，同一角色同一公会同一活动只能领取一次。相关领取状态写入 `guild_activity_claims`，金币与道具发放复用 `currency_logs`、`inventory_items` 和 `drop_logs`。

## 8. 数据存储策略

### 8.1 当前 SQLite 事实数据

SQLite 继续作为不能丢失数据的最终事实来源。

SQLite 存储：

- 账号
- 角色
- 背包
- 装备
- 已学习技能
- 已分配天赋
- 任务进度
- 金币变化流水
- 掉落流水
- 登录日志
- 公会、成员、贡献、商店购买和公会活动领取状态

配置文件存储：

- 地图配置
- 怪物配置
- 装备配置
- 掉落配置
- 任务配置
- 职业配置
- 技能配置
- 天赋配置
- 公会捐献、公会商店和公会活动配置

### 8.2 当前 Redis 缓存与临时状态

v0.3 已接入 Redis，用于保存可重建、临时、高频访问的数据。

Redis 存储：

- 在线状态
- 战斗临时状态
- BOSS 冷却缓存
- 排行榜缓存

会话 token 目前仍以 SQLite 为事实来源；技能冷却和持续伤害属于战斗临时状态，已随在线战斗会话的 `skill_state_json` 一并进入 Redis 会话快照或 SQLite 回退表。限流计数仍属于后续安全增强方向，尚未作为当前 Redis 实现范围。

原则：

- SQLite 保存不能丢的数据。
- Redis 保存可重建、临时、高频访问的数据。

## 9. 部署要求

- 支持 Linux 单机部署。
- 首版使用 Java Spring Boot 单进程服务运行。
- SQLite 数据库文件需方便备份。
- 静态前端资源可由后端服务或 Nginx 托管。
- 后续支持 Docker 部署。

首版推荐运行形态：

```text
game-server
  -> SQLite 数据库
  -> Redis 临时状态与缓存
  -> JSON/YAML 配置
  -> Web 页面
```

首版技术栈：

```text
OpenJDK 8 + Spring Boot 2.7.18 + SQLite + Maven + 原生 HTML/JS
```

## 11. 开发状态与清单

MVP 开发进度以 `doc/development-checklist.md` 为准，后续方向以 `doc/roadmap.md` 为准。

后续每次功能优化或体验优化都必须满足以下前提条件：

- 优化前明确本次优化目标、影响范围和验收方式。
- 每次生成阶段性方案、执行计划或下一步建议时，优先更新 `doc/roadmap.md`、`doc/development-checklist.md` 或本 PRD；确需新增专题文档时，落地后应合并回核心文档或放入 `doc/archive/`。
- 方案落地后，必须同步检查并更新本 PRD 的当前状态、后续方向、页面规划、接口能力或验收标准。
- 优化完成后更新 `doc/development-checklist.md` 的对应任务状态。
- 优化完成后在 `doc/optimization-log.md` 记录优化日期、内容、影响范围、验证结果和后续问题。
- 如果优化改变了产品范围、玩法规则、页面规划、接口能力或验收标准，必须同步更新本 PRD。
- 未完成记录和 PRD 同步前，不视为本次优化闭环完成。
