# v0.5.7 地图点位聚合与日志抽屉计划

## 1. 优化目标

- 解决地图点位在小屏幕和高密度区域互相遮挡的问题。
- 将移动端日志从“整块区域显示/隐藏”升级为底部抽屉，减少玩家来回切换视线。
- 保持地图事件、NPC、怪区和战斗接口不变，本轮继续只优化前端交互承载。

## 2. 范围

- 地图点位按相近坐标聚合：
  - 相距较近的多个点位渲染为一个聚合按钮。
  - 点击聚合按钮展开点位清单。
  - 清单内点击具体点位后复用原有 NPC/事件触发逻辑。
- 移动端底部日志抽屉：
  - 地图页提供底部抽屉，展示交互记录和战斗记录。
  - 抽屉收起时只保留标题和最近提示，展开时查看完整记录。
  - 桌面端仍保留右侧日志栏。
- 静态资源版本号更新，避免浏览器缓存旧脚本和样式。

## 3. 不做范围

- 不新增后端 API。
- 不改变地图点位配置结构。
- 不做真实地图缩放、拖拽视角和后端自由移动。
- 不调整战斗、任务、邮件、GM 等非地图页面逻辑。

## 4. 验收标准

- 同一区域多个点位会合并成聚合按钮，并显示点位数量。
- 点击聚合按钮可展开点位清单，点击清单项可触发原点位动作。
- 移动端地图页可通过底部抽屉查看交互记录与战斗记录。
- 桌面端右侧日志栏仍可正常展示。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
