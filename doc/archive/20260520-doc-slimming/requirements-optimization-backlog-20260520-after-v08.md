# v0.8 后新增优化清单复核 2026-05-20

## 1. 复核结论

> 2026-05-20 更新：本清单中的 v0.9 到 v1.5 主线已按 `doc/v0.9-v1.5-growth-operations-implementation.md` 完成首轮落地；P0-3 战斗与成长数值回归工具已以 `game/scripts/combat-balance-report.js` 形式补齐。

当前版本已经完成 MVP、会话式战斗、Redis 临时状态、邮件补偿、世界 BOSS、可交互地图、任务追踪、格子式背包、背包容量、装备悬浮对比、职业、技能和天赋基础闭环。

复核 `PRD.md`、`api.md`、`game-rules.md`、`development-checklist.md`、旧需求缺口文档和 v0.7/v0.8 方案后，后续重点应从“基础系统是否存在”转向三件事：

- 运营控制补齐：GM 封禁/解封、配置重载和配置校验仍是 PRD 已承诺但未完整闭环的能力。
- 内容密度补齐：地图、怪物、任务已有规模，但装备只有 3 件，长期刷装目标明显不足。
- 构筑深度补齐：职业、技能、天赋已经能跑，但还缺少技能栏、手动施法、技能升级材料、职业专属天赋和数值验证工具。

## 2. P0：优先补齐的系统债

### P0-1 GM 账号状态与配置重载

现状：

- PRD 管理后台明确预留封禁账号和配置重载。
- 当前 GM 已能查角色、发金币、发物品、发邮件、管理地图事件状态。
- `accounts.status` 已存在，但缺少封禁/解封接口和前端入口。
- `GameConfigService` 启动时加载配置，但没有 GM 触发热重载。
- 新增职业、技能、天赋后，配置重载的价值更高，因为调技能/天赋数值不应每次重启服务。

建议范围：

- 新增 GM 封禁/解封账号接口。
- 登录、`/api/auth/me` 和普通业务鉴权统一阻断被封禁账号。
- GM 角色列表显示账号状态。
- 新增配置重载接口，覆盖地图、怪物、物品、掉落、任务、NPC、地图事件、BOSS、世界 BOSS、职业、技能、天赋。
- 重载失败时保留旧配置，并返回错误摘要。
- 重载操作写入 `gm_logs`。

验收标准：

- GM 可以封禁和解封指定账号。
- 被封禁账号不能登录，已有 token 请求也会被阻断。
- 修改 `skills.json`、`talents.json` 或 `classes.json` 后可通过 GM 重载生效。
- 配置错误时接口返回明确错误，旧配置继续可用。

### P0-2 配置引用校验增强

现状：

- 静态配置已经很多，包含 8 张地图、15 个怪物、25 条任务、9 个物品、职业、技能和天赋。
- 技能和天赋已经加入战斗和属性计算，配置错误会直接影响战斗稳定性。

建议范围：

- 校验地图引用怪物、BOSS、NPC、地图事件是否存在。
- 校验掉落表引用的怪物和物品是否存在。
- 校验任务目标、前置任务、奖励物品是否存在。
- 校验技能职业、技能数值范围、冷却、概率、等级、金币消耗。
- 校验天赋前置节点、最大等级、等级门槛、加成字段范围。
- 校验职业基础属性不能为 0 或负数。

验收标准：

- 启动和 GM 重载时执行同一套校验。
- 错误信息能指出配置文件、配置 ID 和字段。
- 校验失败不污染线上已加载配置。

### P0-3 战斗与成长数值回归工具

现状：

- 三职业、12 个技能、6 个天赋已经接入战斗。
- 当前 `mvn test` 能验证编译和基础逻辑，但缺少批量模拟不同职业打怪的数值回归。

建议范围：

- 增加一个开发用战斗模拟脚本或测试类。
- 覆盖 1-8 级三职业、无装备/基础装备/强化装备三档。
- 输出胜率、平均回合、平均掉血、技能触发次数、金币收益。
- 对明显异常场景给出阈值告警，例如低级怪秒杀玩家、高级怪完全打不动、某职业同等级输出过高。

验收标准：

- 一条命令可以跑完三职业基础战斗模拟。
- 输出报告能辅助调职业、技能、怪物和装备。
- 后续改技能/天赋配置时有基础数值护栏。

## 3. P1：提升核心可玩性的新增优化

### P1-1 装备内容扩容与部位补齐

现状：

- PRD 规划装备部位有武器、衣服、头盔、项链、手镯、戒指、鞋子。
- 当前物品 9 个，其中装备只有木剑、布衣、铁剑 3 件。
- 地图和怪物已经扩展到 8 张地图、15 个怪物，但刷装目标不足。

建议范围：

- 补齐头盔、项链、手镯、戒指、鞋子。
- 按地图等级段补普通、精良、稀有装备。
- 新装备补独立像素图标。
- 掉落表按地图和怪物重新分层，让不同地图有明确刷装追求。
- 背包悬浮窗和装备对比继续复用现有规则。

验收标准：

- 每个装备部位至少有 2 件可获得装备。
- 1-8 级地图都有对应掉落追求。
- 装备页可以完整展示 7 个部位的穿戴目标。

### P1-2 强化材料消耗配置化

现状：

- 强化已经消耗金币和材料。
- 当前材料规则仍写在代码或内置逻辑中。
- 已有兽骨、铁矿石、毒囊、骨片、残破符片、赤月尘等材料。

建议范围：

- 新增 `enhancement-rules.json`。
- 支持按强化等级配置金币和材料。
- 第二阶段支持按装备品质、装备等级段或装备部位细分。
- GM 配置重载时校验强化材料引用。
- 前端强化按钮展示下一次强化所需材料。

验收标准：

- 不改 Java 代码即可调整强化金币和材料成本。
- 材料不足时提示缺哪个材料、差多少。
- 高级材料有稳定消耗去处。

### P1-3 消耗品与战斗准备最小闭环

现状：

- 前端已有 `consumable` 类型文案，但配置没有消耗品，后端没有使用接口。
- 当前战斗策略主要来自装备、技能和天赋，缺少战前准备。

建议范围：

- 新增回血药、攻击药、防御药、攻速药。
- 第一版建议战斗外使用，写入“下一场战斗加成”或短时 buff。
- 消耗品占背包格，支持丢弃和出售。
- 在线战斗开始时读取并消耗战斗准备 buff。
- 战斗日志显示药水增益生效。

验收标准：

- 玩家能在背包中使用消耗品。
- 下一场战斗属性得到临时加成。
- buff 使用、过期、消耗都有明确记录。

### P1-4 技能栏与手动施法

现状：

- 当前主动技能自动触发，适合第一版跑通。
- 用户已经希望技能系统“全面一点”，后续需要从自动触发升级到更强的控制感。

建议范围：

- 新增技能栏配置，玩家可选择上阵技能。
- 在线战斗中增加“自动/手动”模式。
- 手动模式下玩家行动轮可选择普通攻击或可用技能。
- 自动模式保留当前逻辑。
- 冷却、持续伤害、治疗仍使用现有 `skill_state_json`。

验收标准：

- 玩家能配置上阵技能。
- 手动战斗可以选择技能释放。
- 技能冷却时按钮禁用并显示剩余回合。
- SSE/轮询模式下状态表现一致。

### P1-5 技能升级材料与技能书

现状：

- 技能学习和升级只消耗金币。
- 背包材料和掉落系统已经成熟，可以承接技能成长。

建议范围：

- 技能学习可选消耗技能书。
- 技能升级从高等级开始消耗材料。
- 不同职业使用不同符文/残页材料。
- 掉落表加入技能书或技能材料。
- GM 可发放技能书用于测试。

验收标准：

- 低级技能仍可金币学习，避免早期卡住。
- 高级升级需要材料，形成刷怪目标。
- 技能页明确展示缺少的材料。

### P1-6 天赋树可视化与职业分支

现状：

- 天赋第一版是紧凑列表，支持前置条件。
- 三职业已完成，但天赋仍是通用节点，职业差异不够明显。

建议范围：

- 前端把天赋从列表升级为分组或轻量树状布局。
- 增加职业分支：战士偏生命/破甲/防御，法师偏爆发/冷却/触发，道士偏持续伤害/治疗/收益。
- 天赋配置增加 `className` 或 `group` 字段。
- 天赋重置可保留，方便平衡测试。

验收标准：

- 不同职业看到或优先展示不同天赋分支。
- 前置关系更直观。
- 加点后角色属性和战斗效果仍正确刷新。

### P1-7 任务目标显式配置与剧情回看

现状：

- 任务接口能返回目标地图和点位，但当前很多任务配置中 `targetMapId`、`targetPointId` 为空，依赖后端推导。
- NPC 对话和地图事件线索多在当次日志里出现，缺少回看。

建议范围：

- `tasks.json` 支持显式 `targetMapId`、`targetPointId`、`guideText`。
- 后端优先使用显式目标，推导作为兜底。
- 新增任务详情抽屉或任务链页。
- 已完成任务可查看剧情摘要、NPC 对话摘要和地图事件线索。

验收标准：

- 核心任务链目标不再依赖模糊推导。
- 玩家能回看已完成任务和关键剧情。
- 地图任务追踪引导文案更稳定。

## 4. P2：中长期玩法扩展

### P2-1 职业转职与职业专属装备

建议范围：

- 高等级开放转职或进阶称号。
- 装备增加职业限制字段。
- 掉落表按职业或通用装备区分。

适合时机：

- 装备部位补齐、技能栏和职业分支天赋完成后再做。

### P2-2 召唤物系统

建议范围：

- 道士 `灵兽契印` 从被动预留升级为召唤物。
- 召唤物有生命、攻击、持续回合或冷却。
- 战斗 action 增加 summon/companion 类型。

适合时机：

- 手动施法和技能状态 UI 稳定后再做。

### P2-3 仓库、整理、装备批量出售

建议范围：

- 仓库容量独立于背包。
- 背包支持排序、按类型筛选。
- 装备批量出售增加二次确认和品质保护。

适合时机：

- 装备数量扩容后，背包压力会明显增加，届时再做价值更高。

### P2-4 拍卖行、公会、PK、运营活动

建议范围：

- 拍卖行承接装备和材料经济。
- 公会承接世界 BOSS 协作。
- PK 承接职业、技能、装备平衡。
- 运营活动承接周期目标。

适合时机：

- 账号安全、GM 工具、日志审计和经济配置稳定后再做。

## 5. 推荐执行顺序

1. `v0.9 GM 账号状态、配置重载与配置校验`
   - 最高优先级，解决 PRD 承诺和后续调数值效率问题。

2. `v1.0 装备内容扩容与掉落分层`
   - 直接补核心刷装目标，收益最大。

3. `v1.1 强化材料配置化与经济调优`
   - 承接装备扩容和材料消耗。

4. `v1.2 消耗品与战斗准备`
   - 增加战斗策略，并复用背包体系。

5. `v1.3 技能栏、手动施法与技能材料`
   - 把技能系统从自动触发升级成玩家可控构筑。

6. `v1.4 天赋树可视化与职业分支`
   - 增强三职业长期差异。

7. `v1.5 任务目标显式配置与剧情回看`
   - 提升任务体验和内容沉淀。

## 6. 本轮暂不建议立即做

- 拍卖行：当前装备和材料总量还不够，做交易系统收益偏低。
- 公会：世界 BOSS 已有，但协作目标还不足。
- PK：职业、技能、装备平衡还没有稳定，不适合过早引入玩家对抗。
- 充值：当前仍是轻量文字页游 MVP，不应抢在玩法闭环前做商业化。
- 跨服：单机部署目标下暂不需要。
