# v0.7.1 天赋系统基础版计划

## 1. 目标

在技能系统稳定后补充长期成长目标。玩家升级获得天赋点，通过天赋节点强化属性、技能触发率、金币收益等方向。

## 2. 设计原则

- 天赋只做加点和属性/战斗修正，不先做复杂天赋树连线 UI。
- 每级从 2 级开始获得 1 点天赋点。
- 天赋节点配置驱动，支持最大等级和前置节点。
- 第一版允许重置天赋，消耗少量金币，方便测试和平衡。

## 3. 首批天赋

- 强健体魄：每级生命 +25。
- 精准打击：每级攻击 +2。
- 铁骨防线：每级防御 +2。
- 疾影节奏：每级攻速 +2。
- 技能熟练：每级主动技能触发率 +2%。
- 战斗嗅觉：每级金币收益 +3%。

## 4. 后端范围

- 新增 `talents.json` 配置和校验。
- 新增 `character_talents` 表。
- 角色响应返回天赋总点数和可用点数。
- 新增天赋列表、加点、重置接口。
- 属性类天赋参与角色属性重算。
- 战斗类天赋参与技能触发率和金币收益。

## 5. 前端范围

- 新增“天赋”页。
- 展示总点数、已用点数、可用点数。
- 天赋节点按紧凑列表展示等级、效果、前置要求和加点按钮。
- 提供重置按钮。

## 6. 验收标准

- 角色升级后可用天赋点增加。
- 天赋加点后属性/战力刷新。
- 前置天赋未满足时不能加后续节点。
- 重置后返还点数并扣金币。
- `mvn test`、前端脚本语法检查通过。
