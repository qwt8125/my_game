# v0.6 背包容量、丢弃与材料消耗基础计划

## 1. 背景

v0.6 第一轮已经完成格子式背包视觉和像素道具图标，但当前背包仍是“无限列表式资产”的业务规则：没有真实容量、没有丢弃、没有批量出售，材料也主要只能卖金币。

本轮已把背包从视觉格子升级为轻量资产管理闭环，为后续装备扩容、材料经济和长期刷装体验打底。

## 2. 目标

- 玩家可以看到背包容量、已用格子和剩余格子。
- 玩家可以丢弃未穿戴装备和材料，避免背包堆积后只能逐个出售。
- 玩家可以批量出售材料，降低重复操作。
- 装备强化从纯金币消耗逐步升级为金币 + 材料消耗。
- 战斗、挂机、任务和邮件附件在满包时有明确处理规则，不吞奖励。

## 3. 推荐范围

### 3.1 背包容量

- 第一版默认容量为 40 格，沿用当前格子视觉数量。
- 装备按每件占 1 格。
- 同类材料按堆叠占 1 格，堆叠上限第一版暂不限制。
- `GET /api/inventory` 返回容量信息：

```json
{
  "capacity": 40,
  "usedSlots": 8,
  "remainingSlots": 32,
  "items": []
}
```

兼容策略：

- 后端可以先增加新的响应包装结构。
- 前端背包页同步适配包装结构，并保留旧数组响应兜底，方便回归。

### 3.2 丢弃接口

新增接口：

```text
POST /api/inventory/{inventoryItemId}/discard
```

请求：

```json
{
  "quantity": 1
}
```

规则：

- 材料可按数量丢弃。
- 未穿戴装备可丢弃。
- 已穿戴装备禁止丢弃，返回明确错误。
- 数量必须大于 0，不能超过当前拥有数量。
- 丢弃需要写入可追踪日志，至少复用或新增资产变化记录，便于排查误操作。

### 3.3 批量出售材料

新增接口：

```text
POST /api/inventory/materials/sell
```

请求：

```json
{
  "items": [
    {
      "inventoryItemId": 50001,
      "quantity": 3
    }
  ]
}
```

规则：

- 第一版只允许材料批量出售，不处理装备批量出售，避免误卖。
- 每个材料仍按 `items.json` 的出售价格结算。
- 金币变化写入 `currency_logs`。
- 前端背包详情区可先提供“出售全部该材料”，批量多选后置。

### 3.4 材料消耗

强化建议从纯金币升级为金币 + 材料：

- 低级装备 +1 到 +3 继续允许只消耗金币或少量兽骨/铁矿石。
- 中后续装备可按品质和强化等级增加材料需求。
- 材料不足时，强化接口返回缺少材料名称和数量。

配置建议：

- 优先在装备或强化规则配置中声明 `materialCosts`，不在业务代码硬编码。
- 第一版可按强化等级配置通用材料消耗，后续再按装备品质细分。

## 4. 满包处理规则

第一版使用“可预期、不会吞奖励”的保守规则：

- 战斗掉落：背包满时，战斗仍可完成；无法入包的掉落通过邮件补发，并在掉落文案中标记“已邮件补发”。
- 挂机领取：如果预计新增物品会超过剩余格子，接口直接阻止领取并提示先清理背包；经验和金币不提前结算。
- 任务奖励：如果任务奖励包含物品且背包空间不足，禁止领取并提示需要空格数。
- 邮件附件：领取附件前检查背包容量；空间不足时不领取附件，不改变邮件领取状态。
- 地图事件奖励：如果奖励包含物品且背包空间不足，阻止触发奖励型事件，避免事件状态已完成但物品未入包。
- GM 直接发物品：空间不足时阻止发放；GM 邮件发放不受影响，玩家领取时再检查空间。

## 5. 前端交互

- 背包顶部展示 `已用 / 容量` 和剩余格子。
- 空格数量由后端容量决定，不再固定写死。
- 物品详情区新增“丢弃”按钮，点击前二次确认。
- 材料详情区新增“出售全部”或数量输入。
- 满包错误用玩家可理解文案展示，例如“背包已满，请先出售或丢弃物品后再领取奖励”。

## 6. 不做范围

- 不做仓库。
- 不做拖拽整理。
- 不做交易、拍卖行或玩家间赠送。
- 不做付费扩容。
- 不做复杂格子形状或多格占用装备。
- 不做装备批量出售。

2. [x] 前端背包页适配容量、已用格子和动态空槽。
3. [x] 新增丢弃接口和前端丢弃确认。
4. [x] 新增材料批量出售能力，第一版先支持“出售全部该材料”。

6. [x] 补满包检查，覆盖任务奖励、邮件附件、挂机领取、地图事件奖励、GM 直接发物品和战斗掉落补发。
7. [x] 更新 `doc/api.md`、`doc/development-checklist.md` 和 `doc/optimization-log.md`。

## 8. 验收标准

- 背包页展示容量、已用格子和剩余格子。
- 背包空槽数量随容量变化，不再只是固定视觉占位。
- 玩家可以丢弃未穿戴装备和材料。
- 已穿戴装备不能丢弃。
- 玩家可以出售某类材料的全部或指定数量。
- 强化装备会按当前内置规则消耗金币和材料，材料不足时返回明确错误；后续可再抽成配置化强化消耗。
- 任务、邮件、挂机和地图事件在背包空间不足时不会吞掉物品奖励。
- `mvn test` 通过；前端脚本语法检查通过。
