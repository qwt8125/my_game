# v0.7 技能系统完整基础版计划

## 1. 目标

在现有会话式回合战斗上补一层可持续扩展的技能系统，让战斗不再只是普通攻击硬拼。第一版参考复古传奇类体验，但采用原创技能名称和描述，避免直接复刻商业游戏文本。

## 2. 设计原则

- 技能配置驱动：新增技能优先改 `skills.json`，少改 Java。
- 职业可限制技能池，但第一版仍允许旧角色平滑迁移。
- 战斗中自动择优触发技能，避免先引入复杂手动操作和 UI 节奏问题。
- 主动技能有冷却、概率、系数和附加效果；被动技能影响角色属性或战斗派生效果。
- 技能升级消耗金币和角色等级门槛，后续可再加入材料消耗。

## 3. 技能分类

### 3.1 战士向

- 烈斩：单体高伤，适合爆发。
- 破甲刺击：中等伤害并临时削弱目标防御。
- 护体战意：被动增加生命和防御。
- 逐风步：被动提高攻速。

### 3.2 法师向

- 火符术：高系数法术伤害，以攻击属性折算。
- 雷光术：中等伤害并附加少量固定伤害。
- 星火燎原：低触发率群攻感技能，当前单怪战斗中表现为高伤害单体爆发。
- 灵能贯通：被动增加攻击。

### 3.3 道士向

- 施毒术：造成直接伤害，并附加持续伤害。
- 回春术：战斗中恢复自身生命。
- 玄灵符：稳定中等伤害。
- 灵兽契印：被动增加生命和攻击，预留后续召唤。

## 5. 后端范围

- 新增 `skills.json` 配置和加载校验。
- 新增 `character_skills` 表。
- 新增技能列表、学习、升级接口。
- 战斗开始时读取角色已学技能，存入战斗会话快照。
- 每次玩家行动时自动从可用主动技能中按触发概率和冷却选择一个技能。
- 技能 action 返回 `action=skill`、技能名、伤害、治疗和描述。
- 被动技能参与角色属性重算，并影响战力。

## 6. 前端范围

- 新增“技能”页。
- 展示可学、已学、等级、冷却、系数、描述、升级消耗。
- 提供学习/升级按钮。
- 战斗日志展示技能释放文本。

## 7. 不做范围

- 不做玩家手动选择技能。
- 不做 PvP 技能平衡。
- 不做召唤物独立单位。
- 不做技能书掉落。
- 不做材料技能升级。

## 8. 验收标准

- 玩家能在技能页看到本职业技能。
- 满足等级和金币时可以学习/升级技能。
- 被动技能学习后角色属性和战力刷新。
- 战斗中主动技能会按概率和冷却自动触发。
- 技能触发、伤害、治疗和持续伤害写入战斗日志。
- `mvn test`、前端脚本语法检查通过。
