## 7. 开发约束

- 所有玩家资产变更必须走数据库事务。
- 密码不能明文存储。
- 配置 ID 使用字符串，例如 `monster_chicken`。
- 玩家动态数据使用数据库自增 ID。
- 不在业务代码中硬编码怪物、地图、装备数值。
- Redis 相关能力已接入临时和高频状态：v0.3 战斗状态已抽象为 `BattleSessionStore`，线上默认使用 Redis 15 号库，SQLite 实现保留为回退模式；在线状态、BOSS 冷却和排行榜使用 Redis 缓存，缓存未命中或异常时回落 SQLite 或持久化事实数据。
- 代码保持 Java 8 兼容，不使用 Java 9+ 语法特性。
- 战斗、掉落、金币、装备变化都要保留日志或流水。
- 在线战斗每次行动推进和最终结算都必须具备幂等保护，避免重复点击造成重复奖励。
- 离线挂机不模拟每个回合，只按时间差后台结算。
