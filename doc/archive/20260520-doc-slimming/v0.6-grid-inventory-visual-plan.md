# v0.6 格子式背包与像素道具图标计划

## 1. 目标

- 将背包页从列表式信息展示升级为类似传奇的格子式背包。
- 每个道具使用独立像素卡通风图标，增强刷装、材料收集和背包整理的游戏感。
- 本轮先做视觉与交互承载，不引入背包容量限制、满包处理、丢弃接口或材料消耗规则。

## 2. 范围

- 背包页改为固定格子面板。
- 已拥有物品按格子展示图标、数量、品质边框和强化等级。
- 空格子展示为空槽，占位用于后续容量规则。
- 保留已有操作：装备可穿戴、强化；全部物品可出售 1 个。
- 为当前 `items.json` 中 9 个物品补充独立像素风图标资产：
  - 木剑
  - 布衣
  - 铁剑
  - 兽骨
  - 铁矿石
  - 毒囊
  - 骨片
  - 残破符片
  - 赤月尘
- 图标统一放在 `game/src/main/resources/static/assets/items/`。

## 3. 不做范围

- 不新增后端背包容量字段。
- 不新增丢弃接口。
- 不新增批量出售。
- 不改变装备强化、穿戴、出售和物品获得规则。
- 不做拖拽整理、仓库、快捷栏或交易。

## 4. 视觉规则

- 单个格子采用与当前白色主题一致的浅色面板、细边框和轻微高光，贴近复古页游背包但不脱离整体界面。
- 图标采用 32x32 像素风 SVG，渲染时保持 crisp edges。
- 品质通过格子边框和小标签区分，不只依赖颜色。
- 数量显示在右下角，强化等级显示在左上角。
- 点击格子后显示物品详情和操作按钮，避免每个格子里塞满文字。

## 5. 验收标准

- 背包页以格子面板展示，不再是普通纵向列表。
- 当前 9 类物品都有独立图标。
- 空背包也显示固定数量空格。
- 点击已有物品格子可查看名称、品质、属性、售价和操作按钮。
- 穿戴、强化、出售 1 个保持可用。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
