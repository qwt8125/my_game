# v0.5.3 地图事件状态管理计划

## 1. 背景

v0.5.1 和 v0.5.2 已完成地图事件奖励、每日重置和每日探索内容扩充。当前事件状态落在 `map_event_states`，但 GM 后台还不能查看或重置某个角色的事件状态，测试、排障和运营调试时不够方便。

## 2. 目标

- GM 可以按角色 ID 查看地图事件状态。
- GM 可以重置某个角色的单个事件状态。
- GM 可以重置某个角色的全部地图事件状态。
- 重置操作写入 `gm_logs`，便于追踪。
- 不新增数据库结构，继续以 SQLite 中的 `map_event_states` 为事实来源。

## 3. 接口设计

- `GET /api/admin/map-event-states?characterId={id}`：返回角色当前已有事件状态，并补充事件名称、地图、类型和配置重置规则。
- `POST /api/admin/map-event-states/reset`：按 `characterId + eventId` 删除单个事件状态。
- `POST /api/admin/map-event-states/reset-all`：删除指定角色全部事件状态。

## 4. 前端设计

- GM 页面新增事件状态管理区。
- 输入角色 ID 后可查询事件状态。
- 每条事件状态提供“重置”按钮。
- 提供“重置全部事件状态”按钮。

## 5. 验收标准

- 非管理员访问接口返回无权限。
- 查询不存在的角色返回明确错误。
- 单个重置后对应事件状态从列表中消失，并可重新触发。
- 全部重置后角色所有地图事件状态清空。
- `mvn test` 和 `mvn package -DskipTests` 通过。
