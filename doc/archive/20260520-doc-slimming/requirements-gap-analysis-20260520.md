# 需求深挖与完善点分析 2026-05-20

## 1. 当前结论

当前版本已经具备完整 MVP、v0.5 地图任务体验闭环和 v0.6 背包资产管理基础闭环：注册登录、角色成长、会话式战斗、掉落、格子式背包、装备、任务、挂机、BOSS、世界 BOSS、邮件补偿、GM 基础能力和可交互地图都已落地。

v0.6 已完成格子式背包视觉、像素道具图标、40 格容量、材料堆叠、丢弃、材料批量出售、强化材料消耗、满包保护、背包悬浮窗、装备对比和已穿戴装备从背包隐藏。继续深挖后，下一阶段不应再把“背包容量/丢弃”作为待办，而应转向“运营控制、装备内容密度、材料经济配置化、战斗准备策略和任务剧情沉淀”。

## 2. P0：PRD 已承诺但仍未闭环

### 2.1 GM 账号状态与配置重载

现状：

- PRD 的管理后台预留包含封禁账号、重载配置。
- 当前 GM 能查询角色、发金币、发物品、发邮件、查/重置/清理地图事件状态。
- `accounts.status` 已存在，但前端和接口没有封禁/解封入口。
- `GameConfigService` 在启动时加载配置，没有 GM 触发重载入口。

缺口：

- 无法封禁账号或解除封禁。
- 登录和 token 鉴权是否严格阻断被封禁账号需要明确。
- 配置改动后需要重启服务，运营调试成本高。

建议：

- 下一轮优先做 `v0.6.1 GM 账号状态与配置重载`。
- 增加账号封禁/解封接口与 GM 前端入口。
- 登录、当前账号、业务鉴权统一阻断已封禁账号，并给出明确错误。
- 增加配置重载接口，重载前后跑引用校验；失败时保留旧配置并返回错误摘要。

### 2.2 文档与规则持续同步

现状：

- PRD 已记录 v0.6 背包容量、悬浮窗、装备对比和已穿戴装备隐藏。
- `doc/api.md` 已记录背包容量、丢弃、材料出售和强化材料消耗。
- 旧需求分析仍把 v0.6 背包规则写成下一步，本次已修正。

缺口：

- 每轮功能快速迭代后，旧路线文档容易继续保留“已完成但仍待做”的表述。
- 规则文档若不同步，后续平衡和配置扩容会参考过时假设。

建议：

- 每轮优化完成后继续同步 PRD、API、开发清单、规则文档和优化日志。
- 旧计划文档可保留历史背景，但需要在顶部或验收段标明实际完成状态。

## 3. P1：核心循环跑通后的可玩性短板

### 3.1 装备内容密度不足

现状：

- 地图 8 张、怪物 15 个、任务 25 条。
- 物品 9 个，其中装备只有 3 件：木剑、布衣、铁剑。
- 装备部位规划有 7 个，但当前只覆盖武器和衣服。

缺口：

- 刷怪后长期追求不足。
- 装备页可用，但装备构筑空间很小。
- 排行榜、BOSS 和世界 BOSS 的战力竞争缺少装备目标支撑。

建议：

- 做 `v0.6.2 装备内容扩容与部位补齐`。
- 至少补齐头盔、项链、手镯、戒指、鞋子。
- 按地图等级段配置普通、精良、稀有装备。
- 同步掉落表，让不同地图有不同刷装追求。
- 为新装备补齐独立像素图标，保持背包体验一致。

### 3.2 强化材料消耗仍需配置化

现状：

- v0.6 已把强化从纯金币升级为金币 + 材料。
- 当前 +1、+2、+3 使用内置材料规则。
- 已有兽骨、铁矿石、毒囊、骨片、残破符片、赤月尘等材料。

缺口：

- 内置强化材料规则不利于后续调数值。
- 高级材料暂时缺少稳定消耗场景。
- 不同品质、等级、部位装备的强化成本还不能区分。

建议：

- 做 `v0.6.3 强化材料消耗配置化与经济调优`。
- 新增强化规则配置，例如按强化等级、装备品质或装备等级段配置 `goldCost` 和 `materialCosts`。
- 配置重载完成后，让强化规则支持热重载校验。
- 让每日探索点产出的材料服务当前地图等级段的装备成长。

### 3.3 消耗品与战斗准备缺失

现状：

- `itemTypeText` 已有 `consumable` 文案，但配置里没有消耗品，后端也没有使用接口。
- 战斗只有属性硬拼，没有药水、临时增益或战前选择。

缺口：

- 玩家战斗策略较少。
- 高难怪物只能靠数值碾压，缺少消耗品调节空间。

建议：

- 做小型消耗品闭环：回血药、攻击药、防御药。
- 第一版可只支持战斗外使用，写入“下一场战斗加成”更合适，因为当前角色没有持久当前生命值概念。
- 消耗品也应占背包材料/道具格，并遵守满包和丢弃规则。

## 4. P1：任务和剧情体验继续深化

### 4.1 任务链专页与剧情日志

现状：

- v0.5.13 已在任务卡上给出推荐和引导。
- NPC 对话和地图事件消息只出现在当次交互日志里。

缺口：

- 玩家看不到已经经历过的剧情线索。
- 多条任务链并行时，缺少“当前章节/故事进度”的总览。

建议：

- 做任务链专页或任务详情抽屉。
- 增加已完成任务回看、NPC 对话摘要和地图事件线索记录。
- 先做前端聚合展示，不急着引入完整剧情编辑器。

### 4.2 任务目标显式配置

现状：

- v0.5.11 后端能推导目标地图和点位。
- 击杀任务目标主要通过怪区包含怪物来反查。

缺口：

- 任务目标推导依赖配置关系，长期内容扩充后容易误判。
- 特殊任务可能需要指定“去 A 地图打 B 怪”，而不是任意包含 B 怪的点位。

建议：

- 在 `tasks.json` 中可选增加 `targetMapId`、`targetPointId`、`guideText`。
- 后端优先使用显式字段，推导逻辑作为兜底。

## 5. P2：更后续玩法方向

### 5.1 多职业与技能

价值：

- 提升角色差异和长期构筑深度。
- 承接 PRD 里的战士、法师、道士方向。

风险：

- 会影响角色创建、战斗公式、装备适配、技能冷却、怪物平衡。
- 不适合作为紧接下一轮，除非先做小型技能原型。

建议：

- 先做 `v0.7 技能最小闭环`，只支持通用主动技能或被动技能。
- 多职业放在技能系统稳定后。

### 5.2 公会、拍卖行、运营活动

价值：

- 公会和拍卖行适合玩家规模上来后增强社交与交易。
- 运营活动适合内容消耗后补充周期目标。

风险：

- 都需要更多账户安全、日志追踪、经济平衡和 GM 工具支撑。
- 当前单机轻量部署阶段，优先级低于 GM 控制、装备内容、材料经济和消耗品。

建议：

- 暂不进入实现。
- 等装备/材料经济稳定后再做拍卖行。
- 等世界 BOSS 和排行活跃后再做公会。

## 6. 推荐下一步路线

本轮复核后，建议后续按以下顺序推进：

1. `v0.6.1 GM 账号状态与配置重载`
   - 这是当前最高优先级。背包资产闭环已完成，下一步应补齐运营控制和配置调试能力。

2. `v0.6.2 装备内容扩容与部位补齐`
   - 拉长刷装追求，支撑排行榜、BOSS 和世界 BOSS 的长期目标。

3. `v0.6.3 强化材料消耗配置化与经济调优`
   - 把当前内置材料消耗转为配置，承接更多材料和装备扩容。

4. `v0.7 消耗品与战斗准备最小闭环`
   - 增加战斗前策略，不急着上多职业和复杂技能。

如果只选一个立即开工，推荐先做 `v0.6.1 GM 账号状态与配置重载`。

## 7. 已完成能力归档：v0.6 背包资产管理

v0.6 背包相关能力已经完成，后续不再作为基础缺口：

- 格子式背包视觉和白色主题适配。
- 每个道具独立像素图标。
- 默认 40 格容量，接口返回 `capacity`、`usedSlots`、`remainingSlots`。
- 材料自动堆叠，装备单件占格。
- 丢弃材料和未穿戴装备。
- 材料批量出售。
- 强化消耗金币和材料。
- 任务、邮件、挂机、地图奖励和 GM 直接发物品的满包保护。
- 战斗掉落满包时通过邮件补发。
- 背包物品悬浮窗和装备属性对比。
- 已穿戴装备不显示在背包中，只在人物装备栏显示。
