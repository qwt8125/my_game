# v0.5.11 任务目标定位元数据计划

## 1. 优化目标

- 将 v0.5.10 的前端轻量目标匹配升级为后端明确返回任务目标地图和点位。
- 降低前端通过名称推断任务目标的误判风险。
- 为后续任务目标高亮、跨地图提示和自动定位打基础。

## 2. 范围

- `GET /api/tasks` 响应新增目标定位字段：
  - `targetMapId`
  - `targetPointId`
  - `targetPointType`
- 后端基于现有配置推导：
  - NPC 对话任务：由 NPC 配置返回所在地图和 NPC 点位 ID。
  - 地图探索任务：由地图事件配置返回所在地图和事件点位 ID。
  - 击杀任务：查找包含目标怪物的怪区或随机遇怪事件点；如果不存在事件点，则回退到包含该怪物的地图。
- 前端任务目标提示优先使用后端定位字段，名称匹配只保留为兜底。

## 3. 不做范围

- 不改任务配置结构。
- 不新增任务目标表。
- 不做跨地图自动切换。
- 不做自动寻路或自动触发点位。

## 4. 验收标准

- 任务接口返回目标定位字段。
- 地图页高亮和“查看目标”优先使用 `targetPointId`。
- 当前地图不包含目标点位时，不展示错误高亮。
- 旧任务展示、领取、推进逻辑不受影响。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
