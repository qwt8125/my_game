# v0.5.13 任务追踪筛选、推荐排序与剧情引导计划

## 1. 目标

- 承接 v0.5.9 到 v0.5.12 已完成的地图任务追踪、目标定位和跨地图提示，把任务追踪从“展示当前任务”升级为“推荐玩家下一步该做什么”。
- 让玩家能在地图页按任务状态快速筛选任务，减少可领取、进行中、锁定任务混在一起造成的判断成本。
- 为任务链补充轻量剧情引导文案，让 NPC 对话、探索点和击杀目标之间更像一条连续路线，而不是零散清单。

## 2. 范围

- 地图页任务追踪面板新增任务筛选：
  - 全部
  - 可领取
  - 进行中
  - 当前地图
  - 跨地图
- 任务追踪排序规则调整为推荐优先：
  - 可领取任务优先。
  - 当前地图可完成任务优先。
  - 主线或前置链靠前。
  - 距离当前等级更近、目标更明确的任务靠前。
- 任务卡补充轻量引导文案：
  - 当前地图目标提示。
  - 跨地图目标提示。
  - 前置未完成时提示需要先完成的任务名称。
  - 可领取时明确提示可以直接领取。
- 任务页与地图页保持同一套状态文案，避免同一个任务在不同页面描述不一致。
- 必要时在任务配置中补充展示文案字段，但不改变任务推进和奖励结算规则。

## 3. 不做范围

- 不新增自动寻路。
- 不自动触发 NPC、地图事件或战斗。
- 不改变任务奖励、进度推进、前置任务判定和领取规则。
- 不引入复杂任务树编辑器或完整剧情系统。
- 不做公会、拍卖行或运营活动配置，这些仍放在更后续版本。

## 4. 建议实现步骤

1. 梳理当前任务响应与前端任务卡渲染字段，确认可复用的状态、目标地图和目标点位信息。
2. 在前端任务追踪面板加入筛选状态与推荐排序函数，优先不新增后端 API。
3. 补充任务卡引导文案，统一地图页和任务页展示。
4. 如现有配置无法表达任务链说明，再为 `tasks.json` 增加轻量展示字段，并同步配置校验。
5. 更新 `doc/PRD.md`、`doc/development-checklist.md`、`doc/api.md` 或规则文档中受影响的部分。

## 5. 验收标准

- 地图页任务追踪可按“全部、可领取、进行中、当前地图、跨地图”筛选。
- 默认推荐排序下，玩家优先看到最值得立即处理的任务。
- 可领取任务、当前地图目标、跨地图目标和前置未完成任务都有清晰文案。
- 任务页和地图页对同一任务的状态描述一致。
- 不影响现有任务推进、奖励领取、NPC 对话、地图事件和战斗结算。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
