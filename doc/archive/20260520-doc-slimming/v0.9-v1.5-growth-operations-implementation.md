# v0.9-v1.5 成长与运营增强落地记录

## 范围

本轮按 `doc/requirements-optimization-backlog-20260520-after-v08.md` 的路线连续落地以下能力：

- v0.9：GM 账号封禁/解封、配置热重载、配置引用校验增强。
- v1.0：装备部位扩容、装备掉落分层、像素图标补齐。
- v1.1：强化规则配置化，支持 +1 到 +5 的金币和材料消耗。
- v1.2：消耗品与战斗准备，药水使用后在下一场战斗一次性生效。
- v1.3：技能材料消耗、技能栏和战斗中手动施法。
- v1.4：天赋扩展为生存、输出、机动三条分支。
- v1.5：任务配置补充剧情说明和行动提示。

## 验收

- GM 后台可以按账号 ID 封禁/解封，并可热重载 JSON 配置。
- 配置热重载失败时不覆盖当前已生效配置。
- 背包支持消耗品堆叠、使用和悬浮窗展示战斗准备效果。
- 战斗开始时消耗已准备的药水加成，并写入战斗日志。
- 技能学习/升级可消耗金币和材料，主动技能可配置到 1-4 号技能栏。
- 玩家回合可手动释放技能；不手动时继续保留自动触发。
- 天赋页按分支展示，天赋前置和点数规则保持有效。
- 任务页和地图任务追踪展示任务剧情与行动提示。
- `scripts/combat-balance-report.js` 可批量模拟三职业 1-8 级对战同等级怪物，并输出异常告警。

## 验证命令

node --check src/main/resources/static/app.js
mvn test
node scripts/combat-balance-report.js
