# v0.5.1 地图事件奖励与每日探索计划

## 1. 背景

v0.5 已完成地图、NPC、任务链、可交互地图、事件状态和移动端基础体验。当前地图事件已经支持一次性事件与短冷却事件，但奖励类型仍偏单一，每日探索点也尚未形成稳定规则。

## 2. 目标

- 地图事件奖励支持金币、经验和物品三类资产。
- 奖励发放必须走数据库事务，并写入金币流水和掉落日志。
- 地图事件支持每日重置，让探索点跨自然日后重新可触发。
- 场景接口继续返回已完成、冷却中、可交互等状态，前端无需来回拖动地图即可识别点位状态。
- 保持现有普通地图、战斗、任务、背包和 BOSS 流程兼容。

## 3. 配置扩展

`map-events.json` 在现有字段基础上新增：

```json
{
  "rewardExp": 15,
  "rewardItems": [
    {
      "itemId": "mat_beast_bone",
      "quantity": 1
    }
  ],
  "resetType": "daily"
}
```

字段规则：

- `rewardGold`：事件奖励金币，已有字段继续保留。
- `rewardExp`：事件奖励经验，可触发升级和战力重算。
- `rewardItems`：事件奖励物品列表，写入背包和掉落日志。
- `resetType=daily`：每日重置事件，触发后到次日 00:00 前不可再次触发。
- 不配置 `resetType` 时沿用现有一次性或冷却逻辑。

## 4. 后端改动

- `MapEventConfig` 增加 `rewardExp`、`rewardItems`、`resetType`。
- `MapInteractionService` 统一处理事件奖励：
  - 金币奖励写入 `currency_logs`，原因使用 `map_event_reward`。
  - 经验奖励通过 `CharacterProgression` 计算升级结果，并更新角色属性。
  - 物品奖励写入 `inventory_items` 和 `drop_logs`，来源使用 `map_event`。
- 每日事件的 `next_available_at` 写为次日零点。
- 场景接口根据 `next_available_at` 判断每日事件是否处于冷却。

## 5. 前端改动

- 地图事件触发成功后继续展示后端 `message`。
- 奖励响应中返回经验、金币、物品、等级变化等字段，后续可用于更细的提示。
- 已完成和冷却状态沿用现有点位样式。

## 6. 验收标准

- 一次性奖励事件仍只能领取一次。
- 短冷却事件在冷却结束后可再次触发。
- 每日事件当天触发后显示冷却中，次日 00:00 后重新可触发。
- 金币、经验和物品奖励到账，并分别写入金币流水、角色数据、背包与掉落日志。
- `mvn test` 通过，`mvn package -DskipTests` 通过。
