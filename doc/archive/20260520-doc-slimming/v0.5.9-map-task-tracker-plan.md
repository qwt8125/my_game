# v0.5.9 地图任务追踪面板计划

## 1. 优化目标

- 让玩家在地图页直接看到当前推荐任务，减少地图页和任务页之间来回切换。
- 在战斗、NPC 对话、地图事件触发和任务领取后，及时刷新地图任务追踪。
- 不新增后端 API，复用现有 `GET /api/tasks` 和 `POST /api/tasks/{taskId}/claim`。

## 2. 范围

- 地图页新增任务追踪面板：
  - 优先展示可领取任务。
  - 其次展示未完成且未锁定的任务。
  - 最多展示 3 条，避免挤压地图。
- 每条任务展示：
  - 任务名称
  - 状态
  - 目标进度
  - 奖励摘要
  - 可领取任务提供“领取”按钮
- 以下操作后刷新任务追踪：
  - 进入地图页
  - 战斗完成
  - NPC 对话完成
  - 地图事件完成
  - 任务领取完成

## 3. 不做范围

- 不新增任务推荐算法。
- 不新增任务自动寻路。
- 不改变任务状态、奖励领取和任务进度后端逻辑。
- 不在地图中直接高亮任务目标点位。

## 4. 验收标准

- 地图页可看到最多 3 条当前任务。
- 可领取任务显示领取按钮，并可直接领取奖励。
- 战斗、NPC 对话、地图事件和领取奖励后，任务追踪能刷新。
- 任务追踪面板在手机端不遮挡地图主体。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
