# v0.5.12 跨地图任务目标提示计划

## 1. 目标

- 让地图页任务追踪在任务目标不属于当前地图时，明确提示目标所在地图。
- 支持玩家从任务追踪卡片一键切换到目标地图，并继续使用已有目标点位高亮与定位能力。
- 复用 v0.5.11 已返回的 `targetMapId`、`targetPointId`、`targetPointType`，本轮不新增后端 API。

## 2. 范围

- 地图页任务追踪卡片新增跨地图提示。
- 目标在其他地图时显示“前往目标地图”操作。
- 切换到目标地图后自动刷新任务追踪，并在能找到目标点位时定位角色前端标记。
- 当前地图内目标仍显示“查看目标”，保持已有交互不变。

## 3. 不做范围

- 不改变角色后端 `currentMapId` 或真实位置事实数据。
- 不做自动寻路、自动触发 NPC/事件或自动战斗。
- 不新增任务配置结构或任务目标表。
- 不改变任务推进、奖励领取、地图事件和战斗结算规则。

## 4. 验收标准

- 当前地图包含任务目标时，任务卡继续显示“查看目标”，点击后高亮并定位目标点。
- 任务目标在其他地图时，任务卡显示目标地图名称和“前往目标地图”按钮。
- 点击“前往目标地图”后，地图视图切换到目标地图，地图列表高亮同步更新，任务目标点位继续高亮。
- 对等级任务等没有地图目标的任务，不显示跨地图操作。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
