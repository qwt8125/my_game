# v0.5.6 地图交互打磨计划

## 1. 优化目标

- 继续改善轻量可交互地图的手机端操作效率，减少地图、日志和战斗记录互相挤压的问题。
- 让玩家可以直接点击地图空白区域移动角色标记，提升地图“可操作”的反馈感。
- 保持当前地图事件、NPC 对话、怪区战斗和任务推进接口不变，本轮只做前端交互承载优化。

## 2. 范围

- 地图页增加快捷操作区：
  - 回到角色：将视角/角色标记恢复到后端返回的当前位置。
  - 显示/隐藏日志：控制右侧交互记录和战斗记录区域。
- 手机端默认收起地图日志区域，优先展示地图本体和点位。
- 点击地图空白区域时，仅移动前端角色标记并追加一条交互记录，不写入后端位置，避免改变当前事实数据模型。
- 点位按钮点击仍按原逻辑触发 NPC、地图事件或战斗，不受空白移动影响。

## 3. 不做范围

- 不新增后端地图自由移动接口。
- 不做完整寻路、碰撞、格子探索或地图拖拽视角。
- 不改变 `currentMapId/currentNodeId/lastX/lastY` 的事实来源。
- 不调整战斗结算、任务推进和地图事件状态规则。

## 4. 验收标准

- 移动端地图页进入后日志区域默认收起，地图本体优先可见。
- 点击“显示日志/隐藏日志”可切换交互记录和战斗记录区域。
- 点击“回到角色”可将角色标记恢复到场景接口返回的位置。
- 点击地图空白处会移动角色标记，并在交互记录中提示“移动到附近位置”。
- 点击 NPC、事件点和怪区仍走原有交互流程。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
