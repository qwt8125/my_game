# v0.5.10 地图任务目标点位提示计划

## 1. 优化目标

- 让地图任务追踪和地图点位产生直接关联，帮助玩家快速找到任务相关目标。
- 在不新增后端 API 的前提下，基于现有任务字段和地图场景字段做前端轻量匹配。
- 保持任务目标提示只是辅助信息，不改变任务推进、NPC、地图事件和战斗规则。

## 2. 范围

- 任务追踪面板新增“查看目标”按钮：
  - 可匹配到目标点位时展示。
  - 点击后移动前端角色标记到目标点位附近，并追加交互记录。
- 地图点位新增任务目标高亮：
  - NPC 对话任务匹配 NPC 点位。
  - 地图探索任务匹配事件点位。
  - 击杀任务匹配怪区或随机遇怪点位。
- 聚合点位中如果包含任务目标，聚合按钮也展示目标态。

## 3. 不做范围

- 不新增后端任务目标点位字段。
- 不做自动寻路或自动触发任务。
- 不改变地图点位配置结构。
- 不在跨地图情况下自动切换地图。

## 4. 验收标准

- 地图页任务追踪中可匹配目标的任务显示“查看目标”按钮。
- 点击“查看目标”会将前端角色标记移动到目标点位附近，并提示目标位置。
- 可匹配任务目标的地图点位有明显高亮。
- 包含任务目标的聚合按钮有明显高亮。
- `node --check game/src/main/resources/static/app.js`、`mvn test`、`mvn package -DskipTests` 通过。
