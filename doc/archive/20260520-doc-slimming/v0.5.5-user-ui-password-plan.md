# v0.5.5 玩家界面信息收口与修改密码计划

## 1. 优化目标

- 清理普通玩家页面中直接暴露的内部字段、配置 ID、英文枚举和值状态，让页面展示更接近玩家语言。
- 登录后增加“修改密码”入口，支持玩家在当前会话内完成密码更新。
- 保留 GM 后台所需的角色 ID、物品 ID、账号 ID、事件类型与状态字段，避免影响运营调试效率。

## 2. 范围

### 前端页面

- 角色页：当前位置展示地图名称或友好兜底，不直接展示 `currentMapId`。
- 任务页：锁定任务展示“完成前置任务后解锁”，不直接展示 `preTaskIds`。
- 背包页：物品品质、类型、装备部位展示中文文案。
- 装备页：装备部位展示中文文案。
- 邮件页：邮件来源展示玩家可理解的来源文案，时间统一做简短展示。
- 战斗记录：战斗状态展示中文文案。
- 顶部全局操作：新增“修改密码”按钮和弹层表单。

### 后端接口

- 新增已登录态修改密码接口。
- 校验原密码正确、新密码长度符合账号密码规则，且新旧密码不能相同。
- 使用 BCrypt 重新写入密码哈希，不保存明文密码。

## 3. 不做范围

- 不调整 GM 后台内部字段展示。
- 不新增找回密码、短信、邮箱或二次验证流程。
- 不强制修改密码后踢出当前登录会话。

## 4. 验收标准

- 普通玩家页面不再直接展示地图 ID、任务前置 ID、邮件 `sourceType`、战斗英文状态、装备部位英文枚举。
- 登录后顶部可打开修改密码弹层，密码修改成功后给出明确提示。
- 原密码错误、新密码过短、两次新密码不一致时给出明确错误。
- `mvn test` 与 `mvn package -DskipTests` 通过。
