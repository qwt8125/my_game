# v0.5.4 地图事件状态清理计划

## 1. 背景

地图事件状态表 `map_event_states` 已承载一次性事件、短冷却事件和每日事件。一次性事件需要长期保留完成状态，但可重复事件在冷却结束很久之后仍保留状态，长期测试或运营后会产生无价值记录。

## 2. 目标

- 保留一次性已完成事件状态，避免玩家重复领取一次性奖励。
- 清理已经过期的可重复事件状态，减少状态表无效记录。
- 提供 GM 手动清理入口，用于测试和小规模运营维护。
- 清理操作写入 `gm_logs`。
- 暂不引入定时任务，避免额外调度复杂度。

## 3. 清理规则

按角色清理满足以下条件的记录：

- `completed = 0`，即不是一次性完成记录。
- `next_available_at` 为空或早于指定截止时间。
- `last_triggered_at` 为空或早于指定截止时间。

默认截止时间为当前时间减 7 天。GM 可在前端传入保留天数，例如 1、7、30。

## 4. 接口设计

```text
POST /api/admin/map-event-states/cleanup
```

请求：

```json
{
  "characterId": 10001,
  "keepDays": 7
}
```

## 5. 验收标准

- 清理不会删除 `completed=1` 的一次性完成状态。
- 清理不会删除仍在冷却中的事件状态。
- 清理会删除超过保留天数的可重复事件状态。
- 清理结果返回删除数量，并写入 `gm_logs`。
- `mvn test` 和 `mvn package -DskipTests` 通过。
