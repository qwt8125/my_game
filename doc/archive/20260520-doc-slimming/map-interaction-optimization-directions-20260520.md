# 地图交互体验优化方向 2026-05-20

## 1. 当前判断

当前地图系统已经具备三栏布局、地图点位、聚合菜单、缩放拖拽、任务追踪、移动端日志抽屉和点击点位触发 NPC/事件/战斗的能力，但体验仍偏“按钮面板”，主要问题是：

- 地图底图主要是网格和灰阶渐变，缺少场景辨识度。
- 点位以矩形按钮呈现，游戏感弱，NPC、怪区、奖励点、传送点的视觉差异不够强。
- 点击点位后角色只是直线移动到目标，缺少路径、方向、行走反馈和到达后的状态承接。
- 操作前缺少预览，玩家点之前不知道会对话、战斗、传送、领取奖励还是冷却。
- 日志与战斗记录承担了太多反馈，地图本体反馈偏少。
- 聚合点能解决重叠，但展开菜单仍像普通列表，和地图空间关系弱。
- 移动端已有抽屉，但点位密集时仍容易误触，地图、任务、日志之间的主次还可以再压缩。

## 2. P0 优先优化

### P0-1 点位视觉升级

- 将矩形点位改成“图标 + 短标签”的地图标记。
- NPC、怪区、探索、奖励、传送分别使用独立像素图标或符号。
- 状态直接体现在点位上：目标、可交互、已完成、冷却、锁定。
- 保留文字标签，但减少按钮感。

### P0-2 点位详情浮层

- 鼠标悬停或点击点位时展示轻量详情浮层。
- 内容包含名称、类型、状态、任务关联、奖励/战斗/传送预期、冷却时间。
- 浮层内放主操作按钮，例如“对话”“探索”“进入战斗”“传送”。
- 地图上直接给反馈，减少玩家依赖右侧日志。

### P0-3 行动反馈增强

- 点击点位后先显示“前往中”状态。
- 角色移动增加方向感、行走节奏和到达闪烁。
- 到达后点位短暂高亮，并在地图附近显示结果 toast。
- 战斗触发前先显示遇敌提示，再进入战斗日志。

## 3. P1 中期优化

### P1-1 场景底图层次

- 根据地图 `backgroundSprite` 或地图类型生成不同视觉层：村外、矿洞、山谷、骨洞、寺院、赤月。
- 使用轻量 CSS/HTML 图层或像素 SVG，不必引入重 Canvas。
- 加入道路、边界、区域块、资源点装饰，让地图不是纯网格。

### P1-2 路径与相邻点位

- 利用 `nextEventIds` 或新增 `linkedPointIds` 画出点位连线。
- 任务目标路径高亮，帮助玩家理解去哪里。
- 可考虑限制只能点击相邻点，或者先只做视觉路线，不改后端规则。

### P1-3 任务与地图联动

- 任务追踪点击后不仅移动角色，也打开目标点位详情。
- 目标点位周围增加脉冲高亮和路径高亮。
- 跨地图任务切图后自动聚焦目标区域。

### P1-4 聚合点优化

- 聚合点展开时用扇形/环形小菜单贴着地图点展开，而不是普通竖向列表。
- 聚合内目标点优先排前，并显示小图标。
- 点击空白区域自动收起聚合菜单。

## 4. P2 后续玩法化

### P2-1 小地图与区域导航

- 右上角增加小地图或地图缩略图。
- 显示当前视口范围、玩家位置、任务目标。
- 点击小地图快速定位区域。

### P2-2 探索雾与区域解锁

- 未探索区域用轻遮罩表现。
- 触发关键事件后解锁新点位或新路线。
- 让地图从“静态菜单”变成“逐步展开的场景”。

### P2-3 战斗入口沉浸化

- 怪区点位可展示怪物头像、推荐战力、主要掉落。
- 点击怪区先打开战斗准备面板，可选择药水、技能栏确认和目标怪物。
- 再进入战斗，减少误触战斗。

## 5. 推荐执行顺序

1. 先做点位视觉升级和点位详情浮层。
2. 再做行动反馈，包括前往中、到达高亮、地图内结果提示。
3. 再做场景底图层次，让每张地图看起来不一样。
4. 最后做路径连线、任务路径和小地图。

第一轮建议控制在前端为主，不改后端数据模型，只复用现有 `type/statusText/description/dialogue/nextAvailableAt/targetPointId` 等字段。
