- v3.7：公会活动与公会排行，`guilds.json` 增加活动目标，新增公会贡献排行、活动领取状态和前端公会活动/排行展示，smoke 覆盖达标领取、防重复领取和排行断言。

### P0.1 前后排技能规则与职业联动（已完成）

### P0.2 装备随机词条与精华材料用途（已完成）

状态：已完成。当前装备生成会按品质写入随机词条，背包与装备页展示词条并提供重铸入口；重铸消耗对应品质装备精华，已穿戴装备重铸后会立即重算角色属性。数值体检已覆盖词条配置、材料引用、掉落等级层级和品质掉率上限。

目标：把装备分解和套装从基础闭环推进到长期刷装追求。

范围：
- 装备生成时支持少量随机词条，例如生命、攻击、防御、攻速或技能触发加成。
- 装备精华增加用途，优先用于词条重铸、词条强化或套装件小额升级。
- 掉落配置补充更清晰的等级层级和品质权重。

验收：
- 背包、装备页能展示随机词条及重铸/强化后的变化。
- 分解产出的精华有明确消耗入口。
- 数值体检脚本覆盖词条范围、材料消耗和掉落引用。

### P2 多人与运营方向

目标：在基础留存和成长追求稳定后，再扩展多人竞争和运营内容。

已完成：
- 排行榜快照元信息：新增 `GET /api/rankings/{type}/snapshot`，前端排行榜展示数据来源、生成时间、刷新间隔和下次刷新时间；旧榜单数组接口继续保留。
- 运营活动配置入口：新增 `activities.json`、`GET /api/activities` 和前端活动页，活动支持状态、时间、奖励预览和目标页面入口。
- 运营活动奖励领取：新增领取状态表、活动奖励领取接口、金币流水/掉落日志写入、背包容量检查和前端领取按钮。
- 运营活动效果系统：新增活动效果配置、配置校验、玩法加成展示，并接入战斗、挂机和世界 BOSS 奖励结算。
- 榜单阶段奖励：活动可按等级榜、战力榜或财富榜当前名次配置阶段奖励，达成后领取并防重复。
- GM 活动编辑后台：管理员可查看并保存活动配置，更新后写回 `activities.json` 并立即生效，smoke 覆盖保存和还原。
- 公会基础闭环：玩家可创建公会并成为会长，未入会玩家可加入公会；会长可踢人和转让会长，最后一名会长退出时公会解散，smoke 覆盖创建、加入、踢人、转让、退出和解散。
- 公会贡献与商店：玩家可按 `guilds.json` 配置每日捐献金币获得贡献；公会页展示个人贡献、公会总贡献、捐献次数和公会商店；商店购买会校验贡献、金币、每日限购和背包容量，并发放道具。
- 公会活动与排行：公会活动按公会总贡献达标，成员可领取配置奖励并防重复；公会贡献排行按总贡献展示名次并标记当前公会。

候选：
- 更高阶公会商店。
- 拍卖行。
- PK 或更多排行榜竞争玩法。
- 排行榜后续可继续升级为 Redis Sorted Set 或定时榜单快照落库。
